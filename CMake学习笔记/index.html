CMake是什么

一个生成工程文件的工具, linux下默认是makefile工程, windows默认是vs工程,
并且，生成工程文件后，可以直接调用make或者vs的编译工具链

扩展一下Linux/windows的编译工具链

Linux工具链

工具名 说明
gcc GNU C 编译器，用于编译 C 程序
g++ GNU C++ 编译器，用于编译 C++ 程序
clang LLVM C 编译器，语法更严格，错误提示更清晰
clang++ LLVM C++ 编译器
make 构建工具，执行 Makefile 中的规则
cmake 构建系统生成器，生成 Makefile、Ninja 或 IDE 工程
ninja 高效构建工具，常与 CMake 配合使用
ld 链接器，负责将 .o 文件链接成可执行文件或库
ar 静态库打包工具，用于生成 .a 文件
strip 去除可执行文件中的符号表，减小体积
objdump 显示二进制文件的汇编、符号、节信息等
readelf 查看 ELF 文件结构和头信息
pkg-config 查询库的编译参数（如 -I, -L, -l）
ldd 查看可执行文件依赖的共享库
gdb GNU 调试器，用于调试 C/C++ 程序
valgrind 内存检查工具，检测泄漏和越界访问

MSVC工具链(Windows)

工具名 说明
cl.exe C/C++ 编译器（MSVC）
link.exe 链接器，生成 .exe 或 .dll
nmake.exe 类似 Unix 的 make，用于执行 Makefile
ml.exe 汇编器（MASM）
dumpbin.exe 查看 .exe 或 .dll 的符号和导出信息
lib.exe 创建或管理 .lib 静态库
rc.exe 编译资源文件 .rc
mt.exe 管理清单文件（manifest）
editbin.exe 修改 PE 文件头（如设置子系统）

其他(Windows)

工具名 说明
cmake 跨平台构建系统，生成 MSVC 工程或 NMake 脚本
ninja 高效构建工具，常与 CMake 配合使用
make GNU Make（通过 MSYS2 或 WSL 提供）
mingw32-make MinGW 的 Make 工具
msbuild.exe 用于构建 Visual Studio 的 .sln 和 .vcxproj，内部也调用 cl 和 link
